# PDF/Image To Editable PPT Design

## 目标

创建一个可复用的 skill，用于将任意 PDF 或图片转换为 PPT，并满足以下优先级：

1. 视觉保真优先：输出 PPT 与原稿逐页视觉一致，保持原底原色，不破坏任何细节。
2. 可编辑元素尽量提取：在不破坏视觉完整性的前提下，尽可能提取文字和图片为可编辑对象。
3. 保守回退：凡是无法高置信重建的元素，保留在底图层，不强拆、不猜测、不牺牲版式稳定性。

## 非目标

- 不承诺任意输入都能 100% 完整还原为全部可编辑对象。
- 不为了提高可编辑率而接受文字错位、字号漂移、行距变化、颜色偏差或图片裁切误差。
- 不把复杂矢量、透明叠加、阴影、滤镜、扫描噪声等高风险元素强行转换为可编辑形状。

## 触发场景

该 skill 应在以下需求中触发：

- 用户要求将 PDF、扫描件、海报、长图或多页文档转换为可编辑 PPT。
- 用户强调保留原样、原版式、原底原色、不失真、不破坏细节。
- 用户要求尽量提取文字和图片，但绝对不能破坏视觉完整性。

## 核心策略

采用双层方案：

- 底图层：每一页先生成高保真整页背景图，保证视觉结果与原稿一致。
- 编辑层：仅对高置信元素叠加可编辑对象。

双层方案的含义是：底图层负责“绝不失真”，编辑层负责“能提则提，但不能破坏底图效果”。如果编辑层与底图层发生冲突，必须优先保留底图层的视觉完整性。

## 输入与输出

### 输入

- 单张图片
- 单个 PDF
- 多页 PDF
- 用户明确要求切分的长页 PDF 或长图

### 输出

- 一个 `.pptx` 文件
- 默认规则为：
  - `PDF 一页 -> PPT 一页`
  - `图片一张 -> PPT 一页`
- 如果用户明确要求切分长页/长图，则允许：
  - `单页/单图 -> 多张 PPT 幻灯片`

## 页面处理流程

对每一页执行相同流程，逐页独立、逐页容错：

1. 渲染高保真页面底图。
2. 分析页面中的文字、图片及其它潜在可编辑元素。
3. 对每类元素做置信度判断。
4. 仅对高置信元素生成编辑层对象。
5. 将底图层与编辑层合成到目标 PPT 页。
6. 若任一元素的重建可能破坏视觉一致性，则取消该元素编辑层输出，保留到底图层。

## 多页 PDF 规则

- 多页 PDF 必须逐页处理。
- 某一页处理失败，不得影响其它页。
- 每页都必须至少生成一张带高保真底图的幻灯片。
- 如果用户未指定切分策略，则严格遵循“每页一张幻灯片”。
- 如果用户指定长页切分，则按内容区段切为多张幻灯片，但每段仍需保留对应底图，避免切分后失真。

## 元素提取策略

### 文字

文字提取必须满足严格门槛：

- 字体可识别或可可靠映射
- 字号可稳定重建
- 颜色可准确提取
- 对齐方式可确定
- 位置可准确映射到 PPT 坐标
- 预计不会导致换行、字距、行距、段落边界明显偏差

只有在以上条件同时满足且总体置信度足够高时，才允许生成可编辑文本框。

只要预计会出现以下任一问题，就不提取该文字为编辑层：

- 位置偏移
- 换行变化
- 字距或行距异常
- 字号不稳
- 颜色不准
- 对齐方式失真

此时应直接保留在底图层，以保持视觉一致。

### 图片

- 能可靠独立提取的图片，应作为单独图片对象放入编辑层。
- 若图片与背景、蒙版、透明效果或其它元素高度耦合，无法可靠分离，则继续保留到底图层。
- 图片提取不得引入裁切错误、缩放失真、透明边缘错误或层级错误。

### 图形与复杂效果

以下内容默认保守处理，不强拆：

- 复杂矢量图形
- 阴影、描边、发光
- 透明叠加、混合模式
- 复杂蒙版
- 扫描噪声与纸张纹理
- 特殊装饰性元素

这类内容优先保留到底图层，除非后续工具链能证明其可高置信重建。

## 失败回退规则

本 skill 的最低成功标准不是“所有内容可编辑”，而是：

- 一定生成可打开的 PPT
- 每页视觉上与原稿一致
- 不因编辑层尝试而破坏页面效果

回退规则如下：

- 元素级失败：仅回退该元素到底图层。
- 页面级失败：该页仅输出底图层。
- 文档级失败：若无法完成完整提取，也至少输出逐页保真的底图式 PPT。

## Skill 结构

建议创建目录：

`skills/pdf-image-to-editable-ppt/`

包含以下内容：

- `SKILL.md`
  - 定义触发条件、处理原则、回退逻辑、输出要求
- `scripts/render_pdf_pages.py`
  - 渲染 PDF 页面到底图图像
- `scripts/extract_images.py`
  - 尝试提取可独立复用的图片资源
- `scripts/extract_text_layout.py`
  - 提取文字块、坐标、字号、颜色、对齐和置信度
- `scripts/build_ppt.py`
  - 生成底图层与编辑层组合后的 PPT
- `references/`
  - 记录依赖、限制、工具链说明

## 依赖策略

skill 采用混合降级策略：

- 优先走 `skill + bundled scripts`
- 如果环境不足，则降级为保真优先流程

也就是说：

- 有脚本依赖时，用脚本完成 PDF 渲染、OCR、版面分析和 PPT 构建。
- 环境不足或依赖不可用时，至少仍应输出保真的底图式 PPT。

## 用户可见承诺

skill 对用户的承诺应精确表述为：

- 尽量提取文字和图片。
- 绝对不能破坏视觉完整性。
- 真正提取为可编辑元素时，必须以视觉一致为前提。
- 做不到高置信重建时，宁可不提取，也不做冒险转换。

## 测试重点

后续实现应至少覆盖以下测试样例：

- 单页海报 PDF
- 多页文档 PDF
- 扫描版 PDF
- 图片型海报
- 长图切分为多页 PPT
- 含复杂透明效果的页面
- 含多栏文本和嵌图的页面

测试重点不是“提取率最大化”，而是：

- PPT 是否逐页视觉一致
- 可编辑文字是否仍保持原位置与版式
- 可编辑图片是否正确分离
- 回退是否足够保守

## 风险与限制

- 某些 PDF 使用子集字体、轮廓字或异常编码，导致文字只能保守回退。
- 扫描件或低清图像中的文字识别可能不足以安全编辑。
- 复杂混合图层无法稳定映射到 PPT 原生效果。
- OCR 与版面分析可辅助提取，但不能覆盖全部高复杂度场景。

本设计明确接受这些限制，只要最终视觉保真不被破坏。
